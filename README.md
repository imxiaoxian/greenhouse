## 📖 Overview

**智慧大棚** is an intelligent greenhouse management system that combines **LangGraph Agent** orchestration, **MCP (Model Context Protocol)** weather services, and a **self-learning pest/disease knowledge base** to provide AI-driven agricultural decision support.

> 🎓 **Academic Context**: Graduation project developed as part of agricultural IoT & AI coursework.

### ✨ Why This Project Stands Out

- 🤖 **Agent-Centric Architecture**: Not just a chatbot — a LangGraph-based state machine with 7 specialized business nodes
- 🔌 **MCP Protocol Integration**: Implements Anthropic's Model Context Protocol for weather tool abstraction (stdio + SSE dual mode)
- 🧠 **Self-Learning Pipeline**: AI-generated pest/disease solutions are automatically vectorized and cached for future recall
- 📊 **Production-Ready Design**: Dual database backends (SQLite for dev, PostgreSQL + pgvector for enterprise), conda environment management

## 🚀 Key Features

### 🌡️ Environmental Monitoring
- **Real-time weather** via OpenWeatherMap API (current conditions + 5-day forecast)
- **Sensor data dashboard** with hour-over-hour change metrics (temperature, humidity, CO₂, light, soil EC)

### 🧠 AI-Powered Optimization
- **Planting recommendations** generated by DeepSeek V4 Pro based on current greenhouse conditions + weather forecast
- **Markdown-formatted** output with actionable temperature, irrigation, fertilization, and ventilation advice

### 🐛 Pest & Disease Intelligence
- **Semantic search** using BAAI/bge-small-zh-v1.5 embeddings (512-dim)
- **Three-tier fallback**: Formal knowledge base → Self-learning cache → DeepSeek real-time generation
- **Structured JSON output** mode for machine-readable treatment plans
- **Auto-learning**: Every AI-generated solution is automatically vectorized and stored for future queries

### 📝 Memo & Notes
- Date-indexed memo system for daily task management
- Filter by date, delete by index

### 📊 Data Analysis (via PyGWalker)
- Interactive drag-and-drop data exploration
- CSV/Excel visualization without writing code

## 🛠️ Tech Stack

| Layer | Technology | Details |
|-------|-----------|---------|
| **Agent Framework** | LangGraph 1.2+ | State machine with conditional routing |
| **LLM** | DeepSeek V4 Pro | langchain-deepseek integration |
| **Embedding** | BAAI/bge-small-zh-v1.5 | 512-dim, local inference via sentence-transformers |
| **Vector DB** | sqlite-vec (default) / pgvector | Lightweight SQLite extension for semantic search |
| **MCP** | FastMCP 1.28+ | Weather tools via stdio/SSE dual transport |
| **Frontend** | Streamlit 1.59+ | 3-page multi-app with st.navigation |
| **Data** | pandas + openpyxl | Sensor data processing & Excel I/O |
| **Env Mgmt** | conda + environment.yml | Project-isolated Python 3.11 environment |

## 📦 Quick Start

### Prerequisites
- Python 3.11+
- conda or venv
- DeepSeek API Key ([platform.deepseek.com](https://platform.deepseek.com))
- OpenWeatherMap API Key ([openweathermap.org/api](https://openweathermap.org/api))

### Installation

```bash
# 1. Clone the repo
git clone https://github.com/imxiaoxian/greenhouse.git
cd greenhouse

# 2. Create conda environment (recommended)
conda env create -f environment.yml
conda activate greenhouse

# Or use pip + venv
python -m venv .venv && source .venv/bin/activate  # Linux/Mac
python -m venv .venv && .venv\Scripts\activate      # Windows
pip install -r requirements.txt

# 3. Configure environment
cp .env.example .env
# Edit .env with your API keys:
#   DEEPSEEK_API_KEY=sk-your-key
#   OPENWEATHER_API_KEY=your-key

# 4. Initialize the knowledge base
python scripts/init_pest_db.py

# 5. Launch the app
streamlit run app/main.py
```

> 💡 **China users**: Set `HF_ENDPOINT=https://hf-mirror.com` in `.env` to download BGE models from HuggingFace mirror.
